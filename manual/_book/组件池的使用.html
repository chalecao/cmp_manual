组件池的使用

组件池的使用

组件池主要用来管理组件，目前支持加载本地组件池和加载网络地址组件池两种形式。 组件池主要是一个组件列表，后缀名为 .cmpp，示例如下：

//index.cmpp
[
  {
    "name": "u-courseCardWithTime",
    "desc": "带有时间的课程卡片",
    "img":"../example/courseCardWithTime/example.png",
    "url": "../example/courseCardWithTime/u-courseCardWithTime.cmp",
    "postUrl": "/api/u-courseCardWithTime"
  },
  {
    "name": "u-liveCard",
    "desc": "直播课时卡片",
    "img":"../example/liveCard/example.png",
    "url": "../example/liveCard/u-liveCard.cmp"
  }
]

其中 name 是组件的标识，desc是组件的描述，img是组件的缩略图，url是加载该组件数据的路径, postUrl是保存组件到组件池时会向改地址提交组件数据。

cmps服务

npm install cmps -g
cmps ./ -p 8000

cmps服务主要是为了方便管理组件。cmps不仅是一个http服务器，而且默认提供了cmp APP应用，默认监听组件池保存组件的接口，自动保存组件到组件配置的url路径上。
